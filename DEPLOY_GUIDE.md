# 🚀 Манифест Деплоя: NEON_RUNNER (GitHub Pages)

Этот документ описывает всю логику работы с ветками сборками и публикацией обновлений игры.

---

## 🏗 Как устроен хостинг Github Pages?

Самая частая ошибка новичков — думать, что GitHub Pages магическим образом сам всё подхватывает. Это не так.
GitHub Pages — это тупой сервер, который раздает то, что лежит в настроенной для него ветке (в нашем случае это ветка **`gh-pages`**). 

**Подхватываются ли изменения из ветки `main` автоматически?** 
**НЕТ.** Ветка `main` (или `r170-experiment`) — это просто твои исходники (черновики). Браузерам нельзя отдавать сырой код, поэтому мы его собираем (компилируем) через Vite.

Каждый раз, когда ты хочешь обновить игру в интернете, тебе нужно вручную прогнать код через `npm run deploy`, чтобы пересобрать папку `dist/` и запушить её в `gh-pages`.

## 📦 Как выкатить обновление для стабильной версии (`main`)

Допустим, ты сидишь в ветке `main` и поменял физику игры, и хочешь выкатить это в сеть.

1. Убедись, что ты находишься в правильной ветке: 
   ```bash
   git checkout main
   ```
2. Проверь локально, что всё работает через `npm run dev`
3. Убей локальный сервер (`Ctrl + C`)
4. Запусти скрипт сборки и публикации:
   ```bash
   npm run deploy
   ```
   *Скрипт пересоберет игру и запишет её в корень ветки `gh-pages`.*
5. Подожди 2-3 минуты и жми `Ctrl + F5` по адресу:
   `https://[твой-ник].github.io/NEON_RUNNER/`

---

## 🧪 Как выкатить обновление для экспериментальной версии (`r170-experiment`)

Ты находишься в ветке `r170-experiment` и решил обновить цвета неоновых платформ.

1. Убедись, что ты в ветке эксперимента: 
   ```bash
   git checkout r170-experiment
   ```
2. Проверь локально через `npm run dev`
3. Убей сервер (`Ctrl + C`)
4. Тебе нужно опубликовать эту сборку в **отдельную папку** (`r170/`), чтобы не удалить основную игру! Для этого запусти:
   ```bash
   npm run deploy
   ```
   *(В файле `package.json` экспериментальной ветки команда уже настроена как `gh-pages -d dist --dest r170 --add`)*
5. Подожди 2-3 минуты и жми `Ctrl + F5` по адресу:
   `https://[твой-ник].github.io/NEON_RUNNER/r170/`

## 🔧 Чеклист Трублшутинга (Ошибки / 404)

1. **Заменил код, задеплоил, а на сайте старьё.**
   - Ты забыл нажать `Ctrl + F5` (или `Cmd + Shift + R`), либо кеш слишком упертый — открой инкогнито (`Ctrl+Shift+N`).
   - GitHub Actions всё ещё собирают сайт в фоне (смотри вкладку "Actions" в репе GitHub).
2. **Бесконечная загрузка "LOADING MODEL" / в консоли 404 по `.glb` моделям.**
   - Убедись, что в файле `src/config.js` пути к моделям начинаются **БЕЗ СЛЭША** (правильно: `assets/models/...`). Со слэшем GitHub Pages будет искать в корне твоего профиля, а не в репозитории.
3. **Белый (или черный) экран, в консоли ругается на `src/main.js` или `style.css`.**
   - Убедись, что в файле `vite.config.js` есть строчка `base: './'`. Без нее Vite склеит пути криво.
